# Agent Skill
**Reference:** [PRD](prd.md)    
This builds on the original PRD to add functionality for an AI agent to interact with the application.

## User Stories & Acceptance Criteria

### User Story: Manage System Users

As an admin user, I want to manage system users so that I can control access to the application.

Acceptance Criteria:

- The user should be able to create a system user.
- The user should be able to edit a system user.
- The user should be able to delete a system user.
- The user should be able to reset a system user's API_KEY.
- The user should be able to disable a system user.
- The user should be able to enable a system user.

### Agent Skill

Provide an agent skill a user can install to their agent to use this application.

### API Authentication

Ensure that APIs needed for the Agent Skill can use API_KEY for authentication.
